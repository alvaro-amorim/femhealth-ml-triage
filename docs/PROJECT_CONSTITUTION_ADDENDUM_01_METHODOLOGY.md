# Adendo 01 — Metodologia de Desenvolvimento Assistido por IA

**Projeto:** FemHealth ML Triage  
**Repositório:** `femhealth-ml-triage`  
**Data:** 2026-06-26  
**Status:** regra complementar ao `PROJECT_CONSTITUTION.md`

## Regra permanente

A partir desta etapa, toda rodada relevante de desenvolvimento, pesquisa, decisão técnica, uso de IA, uso do Codex, validação local, custo/tokens quando disponíveis, testes, commits e problemas relevantes deve ser documentada na trilha metodológica do projeto, para apoiar o relatório técnico final, a rastreabilidade acadêmica e o roteiro do vídeo de demonstração.

## Finalidade

Este adendo cria uma trilha metodológica contínua para registrar como o projeto está sendo desenvolvido, quais ferramentas foram usadas, quais decisões foram tomadas, quais testes foram executados e quais problemas foram corrigidos.

## Arquivos da trilha metodológica

A pasta oficial é `docs/methodology/`.

Arquivos previstos:

- `development_journal.md`: diário cronológico do desenvolvimento;
- `ai_usage_log.md`: registro geral de uso de IA;
- `codex_usage_log.md`: registro das rodadas do Codex;
- `research_log.md`: registro de pesquisas e decisões derivadas;
- `decision_traceability.md`: matriz de rastreabilidade de decisões;
- `cost_tracking.md`: controle de custos e tokens;
- `video_narrative_notes.md`: notas para o roteiro do vídeo.

## Quando atualizar

Atualizar quando houver nova rodada de desenvolvimento, pesquisa relevante, decisão técnica, teste importante, commit, problema/correção ou informação de custo/tokens.

## Uso no relatório e no vídeo

A trilha metodológica deve apoiar uma seção de metodologia de desenvolvimento assistido por IA no relatório final e também fornecer pontos para o roteiro do vídeo.
